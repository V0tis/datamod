# Rin-AI 서비스 확장 로드맵

기본 기능 완료 후, 서비스를 키우기 위한 이성적인 확장 후보입니다.

---

## 1. PDF 내보내기

**목표**: 완성된 리포트 탭의 내용을 PDF로 다운로드 → 리서치 업무 효율 극대화.

**구현 포인트**:
- **대상**: 결과 페이지의 "리포트 요약" 탭 콘텐츠 (뉴스/페인포인트/경쟁사 동향/감정 등).
- **방식**: 
  - 브라우저: `window.print()` + "PDF로 저장" (가장 단순).
  - 또는 `@react-pdf/renderer`, `jspdf` + `html2canvas` 등으로 서버/클라이언트에서 PDF 생성.
- **위치**: 리포트 요약 탭 헤더 또는 카드 상단에 "PDF 다운로드" 버튼.
- **고려**: 인증된 사용자만 허용할지, 공개 결과 페이지에서도 허용할지 정책 결정.

---

## 2. 공유하기 (고유 URL)

**목표**: 특정 리포트의 고유 URL을 생성해 다른 사람에게 공유.

**구현 포인트**:
- **DB**: Supabase `reports` 테이블에 이미 `id`(UUID) 존재 → `/results/[id]` 또는 `/share/[id]` 형태의 공개 URL.
- **옵션 A – 기존 리포트 상세 활용**:  
  이미 `/results/[id]`(또는 `/reports/[id]`)가 있다면, 해당 URL을 "공유 링크"로 복사만 하면 됨.  
  필요 시 `reports`에 `is_public` 또는 `shared_at` 컬럼 추가해 공개 여부 제어.
- **옵션 B – 전용 공유 테이블**:  
  `report_shares (id, report_id, token, expires_at, created_at)` 등으로 일회성/기간 제한 공유.
- **UI**: 리포트 뷰 또는 결과 페이지에 "공유하기" 버튼 → URL 복사 또는 공개 설정 토글.

## 3. 반응 기반 질문 (유저 반응 탭 확장)

**목표**: 유저 반응 탭 하단에서 "이 반응들에 대해 더 궁금한 점이 있나요?"로 추가 질문을 유도.

**구현 포인트**:
- **위치**: 유저 반응(Insights) 탭 콘텐츠 하단에 문구 + 입력창(및 전송 버튼).
- **동작**:  
  - 사용자가 질문 입력 → 기존 유저 반응 텍스트 + 질문을 Gemini에 전달.  
  - 답변은 같은 탭에 이어서 표시(스레드 형태)하거나, 기존 "유저 반응" 블록 아래 "추가 답변"으로 표시.
- **API**:  
  - `POST /api/research/insights/follow-up`  
  - Body: `{ keyword, previousInsights, question }`  
  - 요약본만 전달해 토큰 절약 유지.
- **저장**: 선택 사항. Supabase에 `insights_follow_ups (report_id, question, answer, created_at)` 등으로 남기면 히스토리/재현에 유리.

## 우선순위 제안

| 순서 | 항목              | 이유 |
|------|-------------------|------|
| 1    | 공유하기          | DB 구조가 이미 갖춰져 있어 구현 부담이 적고, 전달 가치가 큼. |
| 2    | PDF 내보내기      | 업무 효율화 수요가 크고, 프론트 중심으로 단계적 구현 가능. |
| 3    | 반응 기반 질문    | 유저 참여·재방문 증가에 기여, Insights API 확장으로 자연스럽게 연결. |

원하시면 위 항목 중 하나를 골라 설계·API·UI까지 구체적으로 구현해 드리겠습니다.
